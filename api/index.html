/contexts

The /contexts endpoints allow access to Apache Spark's SparkContext objects. Each SparkContext is launched in a separate JVM, and is configurable via configuration properties, see the official docs.

/contexts get

get

Lists all current available contexts

get /contexts

Lists all current available contexts

  • Response

HTTP status code 200

Will return an array of context objects

Body

Type: application/json

Example:

{
  "contexts": [
  "my"
  ]
  }

/contexts/{contextName} get post delete

get

Retrieve a specific Spark context

post

Create a new Spark context. If no configuration object is passed in the body, then the context will use the standard values.

Possible properties of the configuration object:

  • createSQLContext (boolean): Whether a sqlContext shall be created with the sparkContext
  • localRepository (string): The URL of a private/local Maven repository
  • masterUrl (string): The master URL for the Spark cluster. Default is local[*]
  • properties (object): The Spark runtime configuration properties to be used.
  • packages (array): An array of package names (Maven notation)
delete

Shuts down a specific Spark context

/contexts/{contextName}/sessions/{sessionId}/statements post

post

Executes (a) statement(s) on an existing session within an existing context

Possible properties of the statement object:

  • code (array<string>): The source code (JS) that should be executed within the opened Spark context. One line should be string one array item
  • return (string): The variable that should be returned from the context, or the code that should be executed to generate the result

post /contexts/{contextName}/sessions/{sessionId}/statements

/contexts/{contextName}/sessions/{sessionId}/dataframes/{dataframeName}/data get

get

Returns data of the DataFrame as a data object, sortable by sortFields and restrictable by start and limit. The result will contain a data property consisting of an array of row objects, as well as an overallCount property, which contains the DataFrame row count.

get /contexts/{contextName}/sessions/{sessionId}/dataframes/{dataframeName}/data

Returns data of the DataFrame as a data object, sortable by sortFields and restrictable by start and limit. The result will contain a data property consisting of an array of row objects, as well as an overallCount property, which contains the DataFrame row count.

  • Request
  • Response

URI Parameters

  • contextName: required (string)
  • sessionId: required (string)
  • dataframeName: required (string)

Query Parameters

  • sortFields: required (string)

    A comma-separated string representation of DataFrame field names. Used to sort the data of the DataFrame (for pagination).

    Example:

    id,name
  • start: required (integer)

    The row number after which the row data from the DataFrame shall be returned.

    Example:

    50
  • limit: required (integer)

    The row count of rows which shall be returned after the start row number.

    Example:

    25

/files

The /files endpoints allow uploading of local files to the Spark-Server. This can be useful if otherwise only the local filesystem can be used (instead of S3 etc.).

/files post get

post

Uploads a file to be used within a Spark computation. The generated fileId can then be used together with the context function getFileById(fileId) (which returns the temporary local path) to actually use the file in the Spark context.

get

Returns an array of uploaded file objects

post /files

Uploads a file to be used within a Spark computation. The generated fileId can then be used together with the context function getFileById(fileId) (which returns the temporary local path) to actually use the file in the Spark context.

  • Request
  • Response

Body

Type: multipart/form-data

Form Parameters
  • file: required (file)

    The file to be uploaded

HTTP status code 201

When the file was successfully uploaded

Body
